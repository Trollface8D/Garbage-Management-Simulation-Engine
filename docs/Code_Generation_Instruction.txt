- While in code generation phase, user must be allow to do these interactions:
a) Cancel button: when click the code generation process will be paused and wait for user to click generate (or resume button, resume button is the same button in the UI, feature as a resume the code generation process when process is paused)
b) Edit button: When click the code generation will be stop and remove all of the progress (same as reset behavior).

Code generation flow
1. User click generate button to start the code generation
2. Enter stage 1 (entities), user has no interaction this this phase. Simulation Entities will be created and referred as target entity. The entire target entity is referred from outside code generation phase called entity list generation, entity in entity list is converted to target entity.
3. After target entity generated, enter stage 1b (policies). User has interactions. First, user must choose the policy that generated from policy entity. Second, user can optionally add policy with title and description. Note, User must choose at least 1 policy to be able to click confirm & proceed button. Chosen policy will be an outline for policy module (present in stage 7) which used for enforcing as a rule inside simulation. 
4. After policies generated, enter stage 1c (dependency). user has no interaction in this phase. This phase must identify dependency between each entity and represent it as inter-entity dependency edge. This edges will determine order of each entity classes (present in stage 2).
5. After dependency generated, enter stage 2 (Entity classes). User has no interaction in this phase. This is iterative phase for creating code generation for each entity classes. Entity classes contain behavior to the other entity, attributes represent entity, etc. 
6. After entity classes generated, enter stage 2v (Validate Protocol). User has no interaction in this phase. This phase must do static checks to ensure that entity classes meet the runtime protocol.
7. After Validated Protocol, enter stage 3 (Environment). User has no interaction in this phase. Environment is created to connect each entity in the defined space which this space can have self behavior that affect other entities.
8. After environment is generated, enter stage 4 (Policy Module). User has no interaction in this phase. Iteratively generate policy module as a rule inside simulation from policy outline, generated from polices stage (stage 1b). Rules can affect entity according to entity's tags or entity's current environment.
9. After policy module is generated, enter stage 4v (Validate policies). User has no interaction in this phase. This phase should make a static check for the each generated policy rule.
10. After policy module is generated, enter stage 5 (Finalize). User has no interaction in this phase. Bundle artifacts (entity_list, manifest, code) for download.
